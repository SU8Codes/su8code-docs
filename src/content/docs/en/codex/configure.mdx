This page is about one thing: **point Codex to SU8 Codes**.

## The two values you need

- Base URL: `https://www.su8.codes/codex/v1`
- API Key: your SU8 Codes key

## Auth header

```bash
Authorization: Bearer <YOUR_API_KEY>
```

## Codex CLI (manual config)

Many Codex CLI versions support configuring providers under `~/.codex/`.

### 1) Open config directory

<Tabs>
  <TabItem label="Windows">
    Create the directory (if needed):

    ```bat
    mkdir "%USERPROFILE%\\.codex"
    ```

    Open it:

    ```bat
    start "" "%USERPROFILE%\\.codex"
    ```
  </TabItem>

  <TabItem label="macOS">
    ```bash
    mkdir -p "$HOME/.codex"
    open "$HOME/.codex"
    ```
  </TabItem>

  <TabItem label="Linux">
    ```bash
    mkdir -p "$HOME/.codex"
    xdg-open "$HOME/.codex" 2>/dev/null || ls -la "$HOME/.codex"
    ```
  </TabItem>
</Tabs>

### 2) Create/modify `config.toml` and `auth.json`

`config.toml` example:

```toml
model_provider = "su8"
model = "gpt-5.2"
network_access = "enabled"
disable_response_storage = true

[model_providers.su8]
name = "su8"
base_url = "https://www.su8.codes/codex/v1"
wire_api = "responses"
requires_openai_auth = true
```

`auth.json` example:

```json
{
  "OPENAI_API_KEY": "<YOUR_API_KEY>"
}
```

> We use the `OPENAI_API_KEY` name for compatibility. It is still your SU8 Codes key.
