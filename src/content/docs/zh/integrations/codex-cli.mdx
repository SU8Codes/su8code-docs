无论您使用的是 **Codex 命令行 (CLI)**、**VSCode 官方插件** 还是 **Codex 桌面平台 (App)**，它们的底层配置文件和运行逻辑都是**完全通用且互通的**。

只需要配置一次底层的鉴权文件，以上三个客户端便全部打通！

## 第一步：安装客户端 (任选其一)

<Tabs>
  <TabItem label="Codex CLI (命令行)">
    如果您偏好终端操作，请先确保机器上安装了 `Node.js`，然后运行：
    ```bash
    npm i -g @openai/codex
    ```
    完成后验证：`codex --version`
  </TabItem>

  <TabItem label="VSCode 官方插件">
    直接在 VSCode 扩展市场搜索并安装 `codex` 插件，或者通过浏览器访问官方地址：  
    [在 VSCode 插件市场查看](https://marketplace.visualstudio.com/items?itemName=openai.chatgpt)
  </TabItem>

  <TabItem label="Codex App (桌面端)">
    您可以前往官方下载独立的桌面端应用，提供全屏视角的项目代码审查与生成体验：  
    [进入 Codex 官方下载引导](https://developers.openai.com/codex/quickstart?setup=app)
  </TabItem>
</Tabs>

---

## 第二步：找到全局配置文件目录

不管您用的 Windows 还是 macOS，这三大官方客户端都会读取系统当前用户目录下的 `.codex` 文件夹。

请打开您的终端或命令提示符（CMD），运行以下命令**直达该目录**：

<Tabs>
  <TabItem label="Windows">
    在 CMD 或 PowerShell 中运行以下命令，会自动在资源管理器中打开该文件夹（如果没有则自动创建）：
    ```bat
    mkdir "%USERPROFILE%\.codex"
    start "" "%USERPROFILE%\.codex"
    ```
  </TabItem>

  <TabItem label="macOS / Linux">
    在终端执行以下命令即可创建并用访达（Finder）打开：
    ```bash
    mkdir -p "$HOME/.codex"
    open "$HOME/.codex"
    ```
    *(如果是 Linux，将 `open` 替换为 `xdg-open`)*
  </TabItem>
</Tabs>

---

## 第三步：写入配置参数

在您刚打开的 `.codex` 文件夹中，新建以下两个核心纯文本文件（不要带 `.txt` 后缀）：

<Tabs>
  <TabItem label="📄 config.toml (指定路由)">
    这个文件告诉所有客户端，我们要把流量定向到 SU8.Codes：
    ```toml
    model_provider = "su8"
    model = "gpt-5.2"
    # 或者换成任何您在模型列表看到的模型，例如 gpt-5.3-codex
    network_access = "enabled"
    disable_response_storage = true
    model_verbosity = "high"

    [model_providers.su8]
    name = "su8"
    base_url = "https://www.su8.codes/codex/v1"
    wire_api = "responses"
    requires_openai_auth = true
    ```
  </TabItem>

  <TabItem label="🔑 auth.json (身份鉴权)">
    这个文件用来存放您在 SU8 控制台生成的令牌：
    ```json
    {
      "OPENAI_API_KEY": "<在这里填入从SU8平台获取的_sk-xxx_令牌>"
    }
    ```
  </TabItem>
</Tabs>

---

> **⚠️ 配置生效必读：**
> - **任何对上述两个文件的修改，都必须完全重启客户端（重启 VSCode，或者在终端 Ctrl+C 杀掉 codex 进程重建）才能生效。**
> - 如果 VSCode 插件一直报错，请先在终端打出 `codex` 测试命令行是否已经成功连通，以便定位问题是出在网络还是 IDE 层。
