import { Steps, Tabs, TabItem } from '@astrojs/starlight/components';

Target outcome: **you have a working Codex client** (CLI or editor extension).

If you can already run `codex` in your terminal, skip to:
- [Configure SU8 Codes](/en/codex/configure/)

## Codex CLI (by OS)

Install success looks like: `codex --version` prints a version.

<Tabs>
  <TabItem label="Windows">
    <Steps>
      1. Install Node.js (LTS)

         ```powershell
         winget install OpenJS.NodeJS.LTS
         ```

      2. Install Codex CLI

         ```powershell
         npm install -g @openai/codex
         ```

      3. Verify

         ```powershell
         codex --version
         ```

      4. Next: configure Base URL + API Key

         - [Configure SU8 Codes](/en/codex/configure/)
    </Steps>
  </TabItem>

  <TabItem label="macOS">
    <Steps>
      1. Install Node.js

         ```bash
         brew install node
         ```

      2. Install + verify

         ```bash
         npm install -g @openai/codex
         codex --version
         ```

      3. Next: configure Base URL + API Key

         - [Configure SU8 Codes](/en/codex/configure/)
    </Steps>
  </TabItem>

  <TabItem label="Linux">
    <Steps>
      1. Install Node.js / npm

         Debian/Ubuntu (NodeSource):
         ```bash
         curl -fsSL "https://deb.nodesource.com/setup_lts.x" | sudo -E bash -
         sudo apt-get install -y nodejs
         ```

         Alternatively, install Node.js LTS via your distro packages or `nvm`.

      2. Install + verify

         ```bash
         npm install -g @openai/codex
         codex --version
         ```

      3. Next: configure Base URL + API Key

         - [Configure SU8 Codes](/en/codex/configure/)
    </Steps>
  </TabItem>
</Tabs>

## Editor extension (e.g. VSCode)

- Extension installed
- You can set Base URL + API Key (or it can read env vars)

> Installation steps vary by version. The key is the next page: point requests to `https://www.su8.codes/codex/v1`.
