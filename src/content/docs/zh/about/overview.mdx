SU8 Codes 做的事情很聚焦：**给 Codex 一个“更稳定、更省心”的入口**。

你可以把我们理解成一个“企业级可用”的网关：
- 你在工具里填 **Base URL + API Key**
- 我们负责把请求稳定转发到 Codex
- 你拿到结果就行（失败也会尽量给到明确原因，方便你排查）

## 我们解决的是什么问题

很多人接入大模型时，最容易踩的坑其实不是“写代码”，而是：
- 网络不稳定、时好时坏
- 工具/插件只支持某种接口（比如只认 `chat/completions`）
- 上线后排错困难（你不知道到底是哪里坏了）

我们希望你接入时更省心、更“像个正经服务”：
- **专业**：给出清晰的接入方式、可复制的配置示例、明确的成功标准。
- **高稳定**：目标可用性 **99.9%**（强调“少出事 + 出事好定位”）。
- **高性价比**：有套餐/余额两种方式，按你的使用习惯选，不用一上来就烧钱试错。
- **工具兼容**：支持 `/responses`，也兼容 `/chat/completions`，很多 AI 编程工具都能接。

## 适合谁用

- 你主要用 Codex，想要一个稳定入口（个人也行，团队也行）
- 你希望“配置一次就能用”，而不是每天和各种 401/404/模型名不对斗智斗勇
- 你要给团队/客户做一个“看起来专业、用起来稳定”的接入

## 你需要记住的两个东西

- Base URL：`https://www.su8.codes/codex/v1`
- 鉴权：`Authorization: Bearer <你的API_KEY>`

> 如果你看到的是 `https://www.su8.codes/...` 这种域名，也不用慌：核心是路径要带上 `/codex/v1`。
