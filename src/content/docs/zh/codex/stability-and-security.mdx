title: 稳定与安全
---

## 稳定性建议
你想要“看起来很靠谱”，最关键的不是写一句 99.9%，而是**客户真的用起来不掉链子**。下面这几条很实用：

- **重试（带退避）**：遇到网络抖动/5xx 时很有用，比如 1s → 2s → 4s。
- **加超时**：别让请求卡死；超时后走重试或提示用户。
- **控制并发**：先从小并发跑稳，再慢慢放大。
- **记录 request id（如果有）**：出问题时好定位，不用靠猜。

## 安全建议
最重要的一条：**API Key 只属于你自己**。

- 别把 API Key 写进代码仓库（尤其别提交到公开仓库）
- 线上环境用环境变量注入 key
- 不要把 key 打进日志里（很多人排错时会不小心打印出来）
