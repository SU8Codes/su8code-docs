title: 常见错误排查
---

## 401
通常是 key 不对，或者 key 没权限。

你可以按这个顺序排：
1. 确认你请求头里真的带了：`Authorization: Bearer <你的key>`
2. 确认你调用的地址是：`https://su8.codes/v1/...`
3. 去后台重新生成一个 key 再试一次（别把旧 key 搞混）

## 404 / model not found
通常是：
- 路径写错（少了 `/v1`）
- 模型名写错

先去这里复制一个模型名再试：
- [模型列表](/zh/tools/models/)

## 429
通常是被限流了。

解决思路：
- 先把并发降下来（一次不要开太多请求）
- 加重试（比如 1s/2s/4s 这种逐步变慢）
- 如果你是脚本循环，记得加一点随机抖动，别整齐地一起冲

## 5xx
通常是服务端临时抖动。

建议：
- 做重试（带退避）
- 给请求加超时（别卡死）
- 如果你每次都在同一时间段报错，建议换个时间再试
