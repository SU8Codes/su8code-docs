su8 codes 现在做的事情很聚焦：**Codex API 中转服务**。

你可以把我们理解成一个“稳定的入口”：
- 你照着我们的 Base URL + API Key 调用
- 我们把请求转到 Codex
- 你拿到结果就行

## 我们解决的是什么问题

很多人接入大模型时，最容易踩的坑其实不是“写代码”，而是：
- 网络不稳定、时好时坏
- 工具/插件只支持某种接口（比如只认 `chat/completions`）
- 上线后排错困难（你不知道到底是哪里坏了）

我们希望你接入时更省心：
- **一个入口更省心**：你只要记住 `https://su8.codes/codex/v1`。
- **兼容常见工具**：既支持 `/responses`，也支持兼容的 `/chat/completions`。
- **上手更快**：文档里有在线测试页，先点一下跑通，再去集成到你自己的系统。

## 适合谁用

- 你主要就想用 Codex（现在我们也主要聚焦在 Codex）
- 你要给团队/客户做一个“看起来专业、用起来稳定”的接入
- 你不想把大量时间花在各种网络和兼容细节上

## 你需要记住的两个东西

- Base URL：`https://su8.codes/codex/v1`
- 鉴权：`Authorization: Bearer <你的API_KEY>`
